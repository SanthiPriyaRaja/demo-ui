🔧 Logout Tenant Reset Fix - Test Instructions

✅ Issue Fixed

Both Login and Register pages now properly reset the tenant dropdown to "Choose your tenant..." after logout instead of defaulting to "Tenant 1 (TechNXT)".

🔍 What Was Changed

  • Login Page: Updated to use useTenant() context instead of getCurrentTenant() utility
  • Register Page: Updated to use useTenant() context instead of getCurrentTenant() utility
  • TenantContext: Added logout event listener to reset tenant state
  • AuthContext: Added custom event dispatch during logout

🧪 Testing Steps

  1. Start the application:
    npm run dev

    Application should be running on http://localhost:5177/ (or next available port)

  2. Test Login Page:
    • Visit http://localhost:5177/login
    • Verify dropdown shows "Choose your tenant..." (not "Tenant 1 (TechNXT)")
    • Select a tenant and login with demo credentials
    • Logout from dashboard
    • Verify dropdown resets to "Choose your tenant..."
  3. Test Register Page:
    • Visit http://localhost:5177/register
    • Verify dropdown shows "Choose your tenant..." (not "Tenant 1 (TechNXT)")
    • Select a tenant and fill registration form
    • After registration, login and then logout
    • Visit register page again and verify dropdown resets

🔧 Technical Implementation

The fix uses an event-driven approach:

  1. AuthContext logout() dispatches custom 'logout' event
  2. TenantContext listens for 'logout' event
  3. When logout event received, currentTenant is reset to empty string
  4. Both Login and Register pages use currentTenant from context
  5. Empty string results in "Choose your tenant..." being displayed
